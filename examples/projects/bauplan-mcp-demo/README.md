# Bauplan MCP Demo

In this repo we use [Claude Code](https://claude.ai/code) with the [Bauplan MCP server](https://docs.bauplanlabs.com) to act as an AI-assisted data engineer. We take stakeholder requests and turn them into production-ready lakehouse pipelines following the **medallion architecture**.


| Layer | Namespace | What lives here |
|---|---|---|
| **Bronze** | `tpch_1` | Raw source tables, read-only (TPC-H dataset) |
| **Silver** | `silver/` | Cleaned, conformed, reference-enriched tables |
| **Gold** | `gold/` | Aggregated, consumer-specific tables |

## How this repo was built

This repo was built incrementally, starting from a minimal scaffold. Nothing in `silver/models.py`, `silver/expectations.py`, `gold/models.py`, `gold/expectations.py`, or the audit bodies in `wap_flow.py` was written by us, all of it was generated by Claude Code across two sessions, each triggered by a single prompt from [`prompts.md`](./prompts.md).

### The initial scaffold

Before any AI interaction, the repo contained only the following files:

| File | State |
|---|---|
| `CLAUDE.md` | Downloaded from the Bauplan skills repo. |
| `lakehouse.md` | Architecture decision guide (what belongs in each layer) |
| `wap_flow.py` | WAP orchestration script with `audit_silver()` and `audit_gold()` as `pass` stubs. This was also generated previously with Bauplan's skills. |
| `silver/bauplan_project.yml` | Project config only! We don´t include any models or expectations. |
| `gold/bauplan_project.yml` | Project config only! We don´t include any models or expectations.|

The architecture and workflow rules were defined upfront so that all AI-generated code would land in the right layer and follow the right constraints automatically.

### Session 1 — build the orders pipeline

> **Prompt:** *"Create a Bauplan data pipeline to answer the following query @ceo_query.md"*

Claude read the CEO query, explored `tpch_1` via the MCP, and applied the medallion rules from `lakehouse.md`. It generated:

- `silver/models.py` — five models: `orders`, `customers`, `nations`, `regions` (bronze→silver base models), and `customers_enriched` (silver enriched model joining customers + nations + regions).
- `silver/expectations.py` — null, uniqueness, and accepted-value tests for every silver table.
- `gold/models.py` — `orders_by_region` aggregating order count, total revenue, and average order value per region.
- `gold/expectations.py` — shape and metric expectations for `orders_by_region`.
- `wap_flow.py` — `audit_silver()` and `audit_gold()` filled in with row-count and region-coverage checks.

### Session 2 — add the customers breakdown

> **Prompt:** *"Number of customers by region and nation"*

Claude recognised that `silver.customers_enriched` already covered this use case and only a new gold aggregation was needed. It added:

- `gold/models.py` — `customers_by_region_and_nation` grouping customer count by region and nation, reading directly from `silver.customers_enriched`
- `gold/expectations.py` — null, accepted-values, and mean-greater-than tests for the new table
- `wap_flow.py` — extended `audit_gold()` with a second query and validation block for the new table

## How it works

```
Stakeholder sends a request like @ceo_query.md
        │
        ▼
Ask Claude Code to create a Bauplan data pipeline with @ceo_query.md
        │
        ▼
Creates models and expectactions for silver and gold layers following the structure defined in `lakehouse.md`
        │
        ▼
Run the WAP flow (wap_flow.py)
  ├── WRITE   — run pipelines on a timestamped branch
  ├── AUDIT   — expectations validate quality at each layer
  └── PUBLISH — merge to main (or leave ready for review)
```

---

## Running the pipeline

The `wap_flow.py` script runs the full bronze → silver → gold pipeline on an isolated data branch, audits each layer, and optionally merges to `main`.

```bash
# Run on a new timestamped branch and leave it for inspection
uv run python wap_flow.py

# Run and auto-merge to main after audit passes
uv run python wap_flow.py --merge

# Run on a specific branch
uv run python wap_flow.py --branch username.my-branch
```
